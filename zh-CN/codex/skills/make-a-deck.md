# 制作演示文稿：HTML 幻灯片演示

将幻灯片演示构建为单个 HTML 文件，使用固定尺寸的幻灯片并在任何视口中自适应信箱化。当用户要求制作演示文稿、演示、幻灯片或提案时使用此技能。

演示文稿是固定尺寸的内容（通常 1920×1080，16:9），缩放以适应任何屏幕。不要手动实现缩放——使用演示文稿外壳起始组件。

## 阶段 1：发现

构建前确认：

- **受众。** 工程师？高管？客户？内部团队？受众决定语气和密度。
- **格式。** 宽高比（默认 16:9，偶尔 4:3）。幻灯片数量（大多数演示文稿在 8–15 张之间）。
- **时长 / 时间预算。** 10 分钟的演示文稿约 10 张幻灯片。30 分钟的约 20–25 张。
- **语气。** 正式商务、轻松内部、营销风格大胆、技术细节。
- **源内容。** 是否有 PRD、文档或现有材料可供参考？在草拟之前阅读它。
- **演讲者备注。** 默认关闭。仅在明确要求时添加。
- **品牌 / 设计系统。** 始终确认。如果没有，在绘制之前调用 `frontend-aesthetic-direction` 技能。

如果用户提供了足够的上下文可以跳过提问轮次（例如，"从这个 PRD 为工程全员大会做一个 5 页的演示文稿"），就继续。

## 阶段 2：规划布局系统

在构建任何幻灯片之前，确定一个布局系统。将其作为注释块写在文件顶部，以便用户看到你的计划。

一个典型的演示文稿有 4–6 种布局类型：

- **封面 / 标题幻灯片** — 大标题、可选副标题、品牌标识、日期
- **章节标题** — 全出血背景色或图片、大章节标题
- **内容幻灯片** — 标题 + 辅助视觉内容（图表、图片或要点列表）
- **引用 / 摘要** — 大引用、归属、大量呼吸空间
- **对比 / 双栏** — 并排内容
- **结尾 / 行动号召（CTA）** — 最终请求、联系方式、后续步骤

对于每种布局，决定：

- 背景颜色或图片样式
- 标题大小和位置
- 正文内容区域
- 页脚处理（页码、品牌标识、无）

限制自己在整个演示文稿中使用 **1–2 种背景色**。章节标题可以换到第三种颜色，但不要更多。

## 阶段 3：构建演示文稿外壳

使用演示文稿外壳起始组件（`copy_starter_component`，`kind: "deck_stage.js"`）。它处理：

- 固定尺寸缩放和信箱化
- 键盘和点击导航
- 幻灯片计数叠加层
- localStorage 持久化（幻灯片索引）
- 打印为 PDF
- `data-screen-label` 和 `data-om-validate` 标签

每张幻灯片是 `<deck-stage>` 元素的直接子元素 `<section>`。

为每张幻灯片添加 `data-screen-label` 属性，以便用户在评论时可以按名称引用：

```html
<section data-screen-label="01 标题">...</section>
<section data-screen-label="02 议程">...</section>
```

**幻灯片标签从 1 开始编号**，以匹配用户看到的幻灯片计数器。

## 阶段 4：逐张构建幻灯片

按顺序逐张构建幻灯片。当你完成 1–2 张时就展示给用户——不要私下完善 15 张再一次性展示。

对于每张幻灯片：

- **排版规则。** 正文文字在 1920×1080 画布上不小于 24px；理想情况 32px+。标题 60–96px+。
- **层级。** 每张幻灯片一个主要信息。辅助元素更小、更低调。
- **图像。** 当幻灯片文字较多时，使用设计系统中的图像，或使用诚实的占位符（条纹背景配等宽标签）。不要用手绘 SVG 插画填充。
- **不要填充。** "为什么选择我们？" / "关于这个演示文稿" / 通用的废话幻灯片浪费用户的注意力。删掉它们。
- **图表和数据。** 展示重要的内容。删除不支持幻灯片要点的列和数据点。

使用设计系统的间距和颜色令牌（或阶段 2 中建立的美学方向）。不要在行内引入新值。

## 阶段 5：演讲者备注（仅在要求时）

演讲者备注默认关闭。仅在用户明确要求时添加。当他们要求时：

- 在头部添加一个 `<script type="application/json" id="speaker-notes">` 数组，每张幻灯片一个条目
- 使用完整的对话脚本，而不是要点大纲
- 带演讲者备注的幻灯片可以在屏幕上显示较少的文字——演讲者填补空白

演示文稿外壳在存在时会自动渲染演讲者备注。

## 阶段 6：验证并交付

在用户的预览中从头到尾浏览演示文稿。检查：

- 在多种视口尺寸下缩放正常
- 幻灯片计数器正确递增
- 键盘导航（方向键、空格）正常工作
- 无内容溢出幻灯片边界
- 字号符合最低要求（正文 24px+）
- 颜色对比度符合 WCAG（进行快速对比度检查，或调用 `accessibility-audit` 进行全面检查）

在回合结束交付时，提供最终的 HTML 文件并亲自运行验证通道 —— 在无头模式下渲染（Playwright 或 chrome-devtools）并检查缩放、布局和溢出。Codex 没有验证器子代理；你需要在循环内完成这个步骤。

简要总结：注意事项（例如，仍需要占位符图像）、后续步骤（例如，需要替换的真实图表）以及用户应签字确认的任何决策。
