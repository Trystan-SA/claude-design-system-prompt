# 前端美学方向：在没有品牌时确定视觉风格

当用户在没有现有品牌或设计系统的情况下进行设计时，建立美学方向（排版、色彩、密度、氛围、组件样式）。在绿地场景下进行高保真工作**之前**使用此技能。

**在没有确定美学方向的情况下从零制作高保真设计，是通向 AI 套路化产出美学的最快路径。** 没有刻意的方向，你会默认选择安全、通用、毫无特色的设计。先确定方向，再在方向内设计。

## 阶段 1：确认确实没有现有上下文

在确定新方向之前，再次检查是否有现有上下文适用：

- 没有品牌指南
- 没有需要匹配的现有应用或产品
- 没有用户明确想要模仿的参考网站
- 代码库中没有部分设计系统

如果以上任何一项存在，**停下来使用它们**。美学方向适用于真正的绿地项目。如果用户有品牌但忘记附上了，在继续之前先要求提供。

## 阶段 2：发现意图

询问用户（或确认他们是否已表述）：

- **三个形容词**来描述想要的感觉。（"编辑感 / 严肃 / 宽敞" vs "活泼 / 大胆 / 张扬" vs "极简 / 安静 / 实用主义。"）
- **受众。** B2B 工程师对美学的反应与消费者或编辑类受众不同。
- **行业背景。** SaaS、消费者、编辑、金融科技、医疗、政府——每个行业有自己的合理默认值。
- **他们欣赏的参考设计。** 具体的品牌、网站或应用。询问他们具体欣赏什么——排版、间距、色彩、调性还是密度？
- **禁区。** 用户明确想要避免的美学或套路是什么？

如果用户不确定，提出 2-3 个以形容词为框架的美学方向，让他们来选择。

## 阶段 3：确定系统——使其具体化

将你的决策以注释块的形式写在用户可以看到的文件顶部。**要具体。** 模糊的美学陈述（"现代而简洁"）会产出通用设计。

在每个轴上做出承诺：

### 排版

选择**具体**的字体（不是"一种无衬线字体"）：

- **标题字体** — 名称、字重、字号梯度
- **正文字体** — 名称（通常是同一字体家族）、字重、字号梯度
- **等宽字体**（如果代码需要）— 名称

避免过度使用的默认字体——Inter、Roboto、Arial、Fraunces、无修饰的系统字体栈。有意图地选择：人文无衬线（Söhne、Suisse）、现代衬线（Tiempos、GT Sectra）、编辑经典（Tiempos Headline、Canela）、打字机等宽（JetBrains Mono、IBM Plex Mono）、几何无衬线（Söhne Buch、Visby），取决于氛围。

如果用户可能无法使用付费字体铸造厂的字体，建议最接近的免费替代（例如，"Inter 已被过度使用，但 Söhne 是付费的——生产环境试试 Söhne，或者用 Albert Sans / Geist 作为免费替代"）。

确定字号梯度（大小、字重、行高）。最多 1–2 个字体家族。

### 色彩

选择一个色调：

- **暖色调** — 奶油、米色、金色、赤土、铁锈
- **冷色调** — 灰色、石板灰、冰色、蓝色
- **中性** — 混凝土、炭色、灰白


然后选择：

- **品牌主色**（带浅色和深色变体）
- **一个强调色**（可选——通常单一强调色就够了）
- **语义色彩**（成功 / 警告 / 错误 / 信息）
- **中性色阶**（5–10 级，从近白到近黑，基于所选色调）

使用 `oklch()` 从零创建调色板时保持和谐：

```css
--brand-primary: oklch(55% 0.18 250);
--brand-accent:  oklch(70% 0.15 30);
```

**微妙地调整白色和黑色。** 纯 `#FFFFFF` 和 `#000000` 显得刺眼。匹配所选色调（例如，`#FAFAFA` 暖中性、`#1A1A1A` 近黑）。

### 密度

选择间距梯度（4px 或 8px 基础）并确定密度：

- **紧凑** — 紧凑仪表盘、密集数据 UI（8px 倍数，较小内边距）
- **正常** — 典型产品 UI（8px 倍数，舒适内边距）
- **宽松** — 编辑、营销、高端感（8px 倍数，大方内边距，大量留白）

密度的选择是感受到的和看到的一样多——它是美学的主要部分。

### 圆角和阴影

- **锐利**（圆角 0–2px）— 实用主义、粗野主义、编辑风格
- **柔和**（圆角 4–8px）— 典型现代产品
- **药丸 / 全圆角**（按钮圆角 9999px；卡片 12–16px）— 活泼、友好、消费者风格

阴影同理：锐利 / 柔和 / 无。确定一个高度系统，不要混用。

### 组件样式

- **填充** — 实底背景按钮，主要操作饱和
- **幽灵** — 无填充按钮，仅边框或纯文字
- **描边** — 有边框，透明填充
- **浮起** — 卡片浮于阴影上

选择一个默认样式，辅助样式用于层级区分。

### 图像和图标

- **摄影** — 真实照片（Unsplash、品牌委托、图库）
- **插画** — 专业素材库或委托创作
- **图标** — Feather、Material、Phosphor、Heroicons，或付费套装
- 当素材不可用时使用**诚实的占位符**

避免手绘 SVG 插画。

### 动效

- **安静** — 最少的动效，仅在状态变化时有过渡（200ms ease）
- **表现力** — 入场动画、滚动驱动展现、视图过渡
- **活泼** — 过冲、弹簧效果、悬停时的微交互

确定一种模式。混合的动效模式会显得毫无意图。

## 阶段 4：在文件中记录方向

将选定的方向写入文件作为可见块——既作为源码顶部的注释，也作为用户可以在渲染输出中看到的"设计系统摘要"部分。就像初级设计师向经理展示他们的思考过程。

示例：

```
/* 美学方向：
 * 编辑感 / 严肃 / 宽敞。
 * - 排版：Tiempos Headline（展示）+ Söhne（正文）— 付费字体铸造厂。
 *   免费替代：GT Sectra → Albert Sans。
 * - 色彩：冷中性。#FAFAFA 背景 / #1A1A1A 文字。
 *   品牌色：oklch(55% 0.18 250) 深蓝。无强调色。
 * - 密度：宽松。8px 梯度，大方内边距。
 * - 圆角：4px（偏锐利）。无阴影——仅边框。
 * - 组件：幽灵按钮。仅主要行动号召（CTA）使用填充。
 * - 图像：真实摄影，全出血。
 * - 动效：安静。200ms ease 过渡，无入场动画。
 */
```

## 阶段 5：应用，然后测试

使用该方向构建一个小界面（一个主视觉区、一张卡片、一组按钮）。尽早展示给用户。

询问："这看起来是否像[你承诺的三个形容词]？"如果答案是否定的——或者用户对特定的轴提出异议——在扩大范围之前修订方向。

在小规模下有效的方向在完整设计中也能支撑住。在小规模下无效的方向在扩大规模时只会变得更糟，而不是更好。

## 阶段 6：一致地使用方向

每个后续设计都应引用方向的设计令牌，而非发明新值。如果新设计需要方向中未定义的值，**先将其添加到方向中**，然后再使用。不要在行内引入一次性值。

最终方向成熟到足以提取到令牌文件时——那就是 `design-system-extract` 发挥作用的时候。

## 阶段 7：总结

报告：

- 三个形容词
- 已确定的排版、色彩、密度、圆角、组件、图像和动效选择
- 在你扩大范围之前用户应审查的任何轴
- 使用该方向构建的第一个界面（链接到文件）
