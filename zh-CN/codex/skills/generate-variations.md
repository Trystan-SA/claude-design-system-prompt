# 生成变体：产出 3 个以上的设计选项

为一个屏幕、组件或流程产出多个截然不同的设计变体，以便用户可以混搭最强的部分。当用户要求选项、替代方案、"不同的方案"，或者他们说"给我看几个"时使用此技能。

**变体是通往好设计的最低成本路径。** 单一设计是一次赌注。三个变体让用户可以拒绝他们不想要的，组合他们想要的。

## 阶段 1：建立基准

确认：

- **什么在被变化。** 单个屏幕、一个组件、整个流程、一种视觉处理？范围决定了有多少变体是有用的。
- **现有设计上下文。** 是否有 UI 套件、设计系统或参考设计？除非明确要求突破，变体仍应扎根于该上下文。
- **变体数量。** 如果用户未指定，默认 3 个。5–6 个是健康上限。超过这个数量用户无法在脑中同时考虑所有方案。
- **轴偏好。** 用户最关注的是视觉、交互、布局，还是文案/调性？你可以在多个轴上变化，但了解用户的优先级有助于权衡探索的方向。

## 阶段 2：选择变化轴

常见的变化维度——选择 2–4 个来进行变化：

- **视觉处理** — 色调（暖 / 冷 / 中性）、密度、阴影样式、圆角、字重
- **布局** — 居中 vs 非对称、单列 vs 多列、全出血 vs 内缩、重网格 vs 流动
- **交互模型** — 单页 vs 多步、弹窗 vs 内联、悬停显示 vs 始终可见
- **信息层级** — 什么被提升，什么是次要的
- **调性** — 极简 / 正式 / 活泼 / 表现力 / 编辑风格
- **组件样式** — 填充按钮 vs 幽灵按钮、带阴影卡片 vs 扁平、圆角 vs 锐利

对于每个变体，写下你正在变化的轴。这使比较对用户来说是可读的。

## 阶段 3：有意图地构建——从基础到大胆

顺序很重要。从最按部就班的开始，以最新颖的结束：

1. **变体 1 — 按部就班。** 匹配现有模式和惯例。"安全"选项。用户知道这行得通，因为它看起来像已经行得通的东西。

2. **变体 2 — 精炼。** 在安全选项的基础上推进一两个维度。相同的整体结构，但更大胆的排版、更自信的布局或更有表现力的色彩选择。这通常是用户的实际选择。

3. **变体 3 — 新颖。** 一个真正不同的方案。可能使用非常规布局、强烈的视觉隐喻、意想不到的交互或大胆的美学。用户可能不会选择它，但它拓展了对话并浮现了他们自己都不知道的偏好。

4. **变体 4–6（如果被要求）。** 光谱上的混合点，或在不同轴上的探索性尝试。

**覆盖两端。** 3 个变体全是"安全"的浪费用户时间。全是"狂野"的则让人感觉你没认真对待简报。

## 阶段 4：实质性地变化，而非表面化地

一个变体不是"相同设计换个颜色"。每个变体应在真正重要的方面有所不同：

✅ 变体之间的差异：布局、层级、什么是主要/次要的、字体系统、密度、交互方式、文案策略
❌ 变体之间相同，除了：按钮颜色、强调色色调、阴影透明度

如果两个变体太接近，删掉一个并替换为更有实质性的替代方案。用户应能用一句话说出任意两个变体之间的区别。


## 阶段 5：在单个文件中呈现

使用 `design_canvas.jsx` 起始组件并排展示静态变体，或者如果变体共享大部分结构且仅在几个轴上有所不同，使用**调整**（参见 `make-tweakable`）。

对于流程或多屏变体，将每个变体构建为画布内的小故事板。

**不要产出 v1.html / v2.html / v3.html。** 一个文件中所有变体可见（或可切换）对于比较来说远比三个独立文件有用。

## 阶段 6：添加注释

为每个变体添加简短说明（一到两句话）：

```
变体 1 — 传统。居中主视觉区，单一行动号召（CTA），高对比度。
变体 2 — 精炼。相同结构，富有表现力的标题字体，更温暖的调色板。
变体 3 — 编辑风格。非对称布局，大型摘要引用，缓慢的滚动驱动展现。
```

说明是一种思考工具。它们迫使你阐明是什么让每个变体与众不同。如果你写不出清晰的说明，说明变体还不够独特。

## 阶段 7：推荐

以明确的推荐结束。用户是决策者，但设计师应给出意见：

- "变体 2 是我的首选——它保留了变体 1 的安全性，但增加了视觉自信。"
- "变体 3 是最有趣的赌注，但对于面向客户的落地页来说风险较高。"
- "变体 1 和 2 很接近——根据你想要中性还是温暖来选择。"

要直接。不要通过说所有选项都同样好来回避。它们不是。

## 阶段 8：交接

用户选择后（或要求再来一轮），建议下一步：

- 如果用户想要精炼所选变体，进行单方向迭代
- 如果用户想继续探索，在不同轴上进行第二轮变体
- `make-a-prototype` 将所选变体做成交互式
- `polish-pass` 如果用户准备好发布所选变体
