# 使其可调整：添加设计内调整控件

为已完成的设计添加一个浮动控制面板，让用户可以实时调整选定的方面——颜色、字体、间距、文案、布局变体、功能开关。当用户想要"尝试不同选项"、"看看不同版本"或并排比较视觉选择时使用此技能。

**一个文件，多个变体。** 调整功能让单一设计可以承载多条视觉探索路径，而无需在项目中散布 v1.html / v2.html / v3.html。

## 阶段 1：确定哪些应该是可调整的

与用户确认——或提出建议并检查——应暴露哪些方面。常见候选项：

- **颜色** — 品牌主色、强调色、背景色调
- **排版** — 字体家族、基础字号、比例系数
- **密度** — 间距梯度（紧凑 / 正常 / 宽松）
- **布局变体** — 居中 vs 左对齐、单列 vs 多列
- **组件变体** — 按钮样式（填充 / 幽灵 / 描边）、卡片处理
- **文案** — 标题、副标题、行动号召（CTA）文本
- **功能开关** — 显示/隐藏推荐评价区、显示/隐藏页脚注册区等

抵制暴露每一个可能的旋钮。**保持调整面少** — 3–8 个控件是健康的范围。目的是给用户几个有意义的轴来探索，而不是重建 Figma。

如果用户没有要求调整但设计有明显的变化轴，默认添加 1–2 个来展现有趣的可能性。

## 阶段 2：设计调整面板

调整面板位于**原型内部**，在一个浮动面板中——通常在右下角，半透明，带有微妙的边框。标题命名为 **"调整"** 以匹配工具栏切换按钮的命名。

每个控件应该是与值匹配的正确类型：

- **颜色** → 颜色选择器
- **字体 / 字体家族 / 变体** → 下拉菜单或按钮组
- **数值（字号、间距）→ 滑块**，带合理的最小/最大值
- **布尔值（显示/隐藏区域）→ 开关
- **文案** → 文本输入框或文本区域

保持控件紧凑。紧密堆叠的一列控件比一个铺开的面板更好。

## 阶段 3：连接实时更新

使用 CSS 自定义属性作为视觉令牌——它们会更新所有引用它们的内容：

```css
:root {
  --tweak-primary: #0066CC;
  --tweak-font: "Inter", sans-serif;
  --tweak-density: 16px;
}
```

当调整值变化时，更新自定义属性：

```js
document.documentElement.style.setProperty('--tweak-primary', newColor);
```

对于非 CSS 值（文案、布局变体、功能开关），使用 JS 状态配合重新渲染或 DOM 操作。

## 阶段 4：实现宿主协议

宿主环境暴露一个工具栏切换按钮来显示/隐藏调整面板。连接协议使切换按钮工作：

1. **首先在 `window` 上注册 `message` 监听器** — 在宣布可用性之前：
   - `{type: '__activate_edit_mode'}` → 显示你的调整面板
   - `{type: '__deactivate_edit_mode'}` → 隐藏它
2. **然后宣布可用性：**
   ```js
   window.parent.postMessage({type: '__edit_mode_available'}, '*')
   ```
3. **当值变化时，通过回发持久化：**
   ```js
   window.parent.postMessage({type: '__edit_mode_set_keys', edits: {primaryColor: '#FF6600'}}, '*')
   ```
   你可以发送部分更新——只有你包含的键会被合并。

**顺序很重要：** 如果你在注册监听器之前宣布 `__edit_mode_available`，宿主的激活消息会在你的处理器存在之前到达，切换按钮会静默无响应。

## 阶段 5：将默认值持久化到磁盘

将你的可调整默认值包裹在注释标记中，以便宿主可以重写它们：

```js
const TWEAK_DEFAULTS = /*EDITMODE-BEGIN*/{
  "primaryColor": "#D97757",
  "fontSize": 16,
  "dark": false
}/*EDITMODE-END*/;
```

标记之间的块**必须是有效的 JSON**（双引号键和字符串）。根 HTML 文件中必须恰好有一个这样的块，位于内联 `<script>` 中。宿主会将你的编辑合并到此块中并写回文件，因此更改在重新加载后仍然保留。

## 阶段 6：关闭时隐藏控件

当调整功能关闭时，设计应看起来是最终成品。面板应完全隐藏——不仅仅是变暗，不仅仅是折叠到角落。用户应看到设计是一个精致的成品，没有可见的调整界面元素。

这是不可协商的：当调整功能关闭时留下任何可见的"编辑模式 UI"会让设计看起来未完成。

## 阶段 7：验证

在用户的预览中：

- 通过工具栏切换面板的显示和隐藏——干净利落
- 更改每个调整项——实时更新
- 重新加载页面——调整后的值保持不变
- 在面板关闭的状态下检查设计——它看起来像一个完成的设计，没有可见的调整界面元素

## 阶段 8：总结

报告：

- 暴露的调整项及其值类型
- 默认值
- 你考虑过但排除的调整项（及原因）
- 调整集是否覆盖了用户预期的探索轴
